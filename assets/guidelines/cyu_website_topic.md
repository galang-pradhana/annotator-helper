PENJELASAN SINGKAT SERTIFIKASI KE- 4 CYU WEBSITE TOPIC
1. CYU Website Topic ini tentang apa ?
Bayangkan kamu sedang asyik browsing di internet dan membuka banyak tab. Kadang kita bingung sendiri tab mana yang bahas apa. Nah, fitur CYU Website topik ini gunanya seperti "tukang label otomatis".
Sistem AI (LLM) akan mengambil 3 website yang kamu buka (yang disebut satu Cluster) dan mencoba merangkumnya menjadi satu Topik pendek (biasanya kurang dari 3 kata) supaya kamu gampang merapikan dan mencari kembali tab-tab tersebut.
Contoh Gampangnya: Kalau kamu buka 3 tab tentang "obat flu alami", "cara sembuh dari pilek", dan "makan sup saat flu", AI harusnya memberi label: "Pengobatan Flu".

--------------------------------------------------------------------------------
2. Bagaimana Cara Penilaiannya? (Analogi & Logika)
Tugas kamu adalah menjadi "Quality Control" atau juri untuk mengecek apakah label yang dibuat AI itu sudah pintar atau malah ngawur. Berikut alur penilaiannya dengan logika yang super simpel:
Langkah 1: Cek "Barang"-nya (Review Original Input)
Sebelum menilai AI, kamu harus baca dulu 3 website itu (Judul dan cuplikan teksnya).
Logikanya: Kamu harus tahu apa isi "kotak"-nya sebelum bisa menilai apakah "label"-nya benar.
Tugasmu: Cari Core Idea (ide utama/siapa tokoh utamanya) dan Key Detail (info penting tambahan).
Langkah 2: Cek Apakah Isi Kotaknya Nyambung? (Consistency)
Kamu harus cek apakah 3 website tadi sebenarnya membicarakan hal yang sama atau tidak.
Analoginya: Kalau dalam satu kotak ada "Apel", "Jeruk", dan "Sepatu", berarti isinya nggak nyambung (tidak konsisten).
Penilaianmu: Apakah judul dan teks di 3 web itu punya "benang merah" yang sama?.
Langkah 3: Menilai Label dari AI (Evaluate Responses)
Ini bagian paling penting. Kamu menilai label (Topik) yang dibuat AI berdasarkan beberapa poin:
1. Ketepatan (Comprehensiveness):
Apakah mencakup ide utama? Misal, webnya bahas film Conclave, tapi AI kasih label "Proses Pemilihan Paus". Itu salah, karena topik utamanya adalah filmnya, bukan proses agamanya.
Jangan terlalu umum, jangan terlalu detil. Misal, web bahas "Situs Walmart Down", AI cuma kasih label "Walmart". Itu terlalu umum (vague). Harusnya "Masalah Situs Walmart".
Nama Penting Harus Ada: Kalau 3 web itu semuanya bahas tokoh bernama "Klaus Schwab", maka namanya wajib ada di label. Jangan cuma disebut "Ketua Organisasi".
2. Kebenaran Fakta (Groundedness):
Analoginya: AI dilarang jadi "tukang gosip".
Logikanya: AI hanya boleh membuat label dari kata-kata yang ada di teks. Kalau AI menambah informasi baru hasil imajinasinya sendiri (halusinasi), maka nilainya buruk.
3. Kerapihan Bahasa (Composition):
Apakah bahasanya enak dibaca dan tidak aneh?.
4. Kepuasan (Satisfaction):
Secara keseluruhan, sebagai pengguna, kamu terbantu nggak dengan label itu? Kalau labelnya bikin kamu cepat nemu tab yang dicari, berarti Highly Satisfying.
Ringkasan Logikanya:
Lihat isinya (3 website).
Cek nyambung nggak isinya satu sama lain.
Cek label AI-nya: Pas tidak atau ngarang (ngawur).

CYU - WEBSITES TOPIC (GUIDELINES)
Version 25.05.12. May 2025
 CYU - Regular Summarization Guidelines
Before using this guideline, make sure you have read and are familiar with CYU - Regular Summarization Guidelines (previously Summarization Human Eval 4.0 Guidelines) which includes the complete workflow of general CYU (Catch You Up), or Summarization, tasks.
This CYU - Websites Topic guideline is tailored specifically for evaluating Website topic-style summary and should be treated as an addendum to the CYU - Regular Summarization Guidelines.
Introduction
This new project is meant to support a new CYU (Catch You Up) or Summarization feature. We'll use the code name Navy Zinc to refer to this feature throughout the guidelines.
Feature Overview
Navy Zinc supports a feature that groups webpages by semantic value and gives them a topic to help users keep organized.
 The topic is generated by an LLM from 3 grouped websites—together these are called a Cluster.
Example: If a user searches for different cold remedies with three or more websites open, a cluster algorithm will group sites that are about colds. Navy Zinc topic organizer will then summarize the websites and give it a topic such as "Treatments for Cold".
Cluster Example
:
a. A deeply frustrating line-by-line reading of Casey Costello's letter to Health NZ | The Spinoff
b. Coalition directs Health NZ to stop saying 'pregnant people' | RNZ News
c. casey costello - Google Search
 Topic: Casey Costello
Task Goal
The goal of this task is to evaluate whether the LLM-generated topics are correct and relevant to the user.
 You will follow a workflow similar to that of CYU - Regular Summarization, with questions specifically designed for CYU - Websites Topic.
 For this project, Comprehensiveness needs your special attention. The questions are significantly different from the regular Summarization tasks.
Key Terms
Term
Definition
Website
A website consists of a website title and feature text.
Website Title
The title of a webpage. Example: "aldi product recall - Google Search".
Feature Text
A snippet of text that previews or describes the webpage content.
Cluster
A group of 3 websites. For this task, you will assess the quality of the cluster to determine if all websites share the same core idea.
Core Idea
Refers to a core attribute or the most important aspect of something. The major subject or entity in a website (website title + feature text).
Key Detail
A phrase or word that describes or gives additional information about the core idea. It's what makes the core idea distinct. Example: "recall".
Summary Point
For Websites Topic task, a "summary point" refers to the individual point captured in a response.
Note: Not all topics would include both core idea and key details, given how short a topic can be (less than 3 words). In many cases, a core idea itself is sufficient.

Step 2: Evaluate the Original Input Text
Safety / Harmfulness
Question: Does the input text contain content that is harmful, unsafe, or should be handled with care?
Scale
Definition
Harmful/unsafe
Confident the content is harmful or unsafe.
Maybe harmful/unsafe
Content may be harmful in wrong context; handle with care.
Not harmful/unsafe
Confident content is harmless or safe.
Irregularity
Question: Does the input text contain any irregularities that suggest the text was artificially assembled rather than organically created?
Scale
Definition / Issues
Yes
Text contains irregularities: Formatting issues (spacing, codes), Content inconsistency (title/text mismatch), Naming inconsistencies (different entities), Missing content.
No
Text doesn't include irregularities.
Consistency
Determine if websites share the same core idea.
Scenario: not_consistent rating
 If both Consistency questions below are "No":
A high-level summary point: Model may generate a broader theme (e.g., Day of the Dead, Oktoberfest, Diwali -> "cultural event"). This is considered Good and Comprehensive.
Inadequate summary point: If no broader theme exists and the point is irrelevant, indicate poor result under Comprehensiveness.

Yes: All titles share same core idea.
No: No coherent identifiable core idea across titles.
Question 2: Do all website feature texts share the same core idea?
Yes: All feature texts share the same identifiable core idea.
No: No coherent identifiable core idea across feature texts.

--------------------------------------------------------------------------------
Step 3: Evaluate the Responses
Instruction Following
Auto-graded. Checks if the response is within 3 words.
Comprehensiveness
Question 1: Does the summary point correctly cover the core idea shared by all websites?
Yes: Includes Broader Themes (e.g., Prozac + Effexor -> "antidepressants").
No: Misses core idea. (Example Cluster: Conclave movie info. Label: "Pope Selection Process". Rating: No, should be "Conclave").
Question 2: Does the summary have the appropriate level of key details shared in all websites?
Yes: Appropriate detail for organization.
No - irrelevant detail: Overly specific/non-universal. (Example: "Accountant 2 Box Office Potential" when websites are just movie reviews).
No - vague: Too broad. (Example: "Walmart" instead of "Walmart Outage").
Question 3: Does the summary point capture the entity (name of person, location, institute, landmark) if shared in all websites?
Yes: Captures the entity.
No: Misses the entity. (Example: "WEF Chairman" instead of "Klaus Schwab").
Not Applicable: No shared entity exists.
Groundedness
Evaluates if response adds or assumes info not in text.
Yes: Based strictly on text or logical inferences.
No: Hallucination (new info), Inaccuracy (contradiction), False Connection (linking unrelated text).
Hallucination Example: Adding "Research" to a cluster about "Instructor Salaries".
False Connection Example: Linking "Alumni" and "Glassdoor" as "Alumni at Glassdoor" when they are independent facts.
Composition
Question 1: Localization issues. (Yes/No).
Question 2: Easy to understand. (Yes/No). If you need to repeatedly check original text, it is not easy to understand.
Satisfaction
Holistic rating for effectiveness in organizing and locating websites.
Scale
Definition
Highly Satisfying
Captures Core Idea + Key Detail; specific; Grounded; Not Harmful.
Slightly Satisfying
Requires evaluator judgment; functional but perhaps imperfect.
Slightly Unsatisfying
Relevant but lacks precision; not very useful for quick location.
Highly Unsatisfying
Total misinterpretation; Not Grounded; or Harmful.

--------------------------------------------------------------------------------
COMPARATIVE SATISFACTION RATING FRAMEWORK
Rating komparasi murni melihat nilai Satisfaction saja (mengabaikan Instruction Following, Groundedness, dsb secara terpisah).
Logika Perbandingan:
Jika response sama: Same.
Jika naik/turun 1 tangga: Slightly Better / Slightly Worse.
Jika naik/turun 2 tangga: Better / Worse.
Jika naik/turun 3 tangga: Much Better / Much Worse.
Contoh Tangga Nilai:
A Highly Satisfying vs B Highly Unsatisfying = A Much Better.
A Highly Satisfying vs B Slightly Unsatisfying = A Better.
A Highly Satisfying vs B Slightly Satisfying = A Slightly Better.
A Slightly Satisfying vs B Highly Unsatisfying = A Better.
